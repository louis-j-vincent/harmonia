What "0% at offset 0" vs "25% at offset +12" Means

The Phase 2 analysis was testing whether inferred chords match GT roots when compared at different bar offsets:

  • Offset 0: Assuming chart position is correct, compare inferred to GT
  • Offset +12: Shift comparison window 12 bars later and recompare

If accuracy jumps from 0% to 25% when offset changes, it suggests either:

  1. The GT position is wrong by 12 bars (but user says no)
  2. Time alignment between GT and inferred is off by ~12 bars worth of time
  3. The model's predictions systematically lag behind by ~12 bars worth of content

Hypothesis About "0% at offset 0"

When the previous analyst tested "offset 0" for section B specifically, they likely:

  1. Extracted GT B section roots: [A, D, G, G, C, F, A#, D#]
  2. Extracted inferred chords from the same time window
  3. Compared them directly
  4. Found 0 matches

Likely cause: The inferred chord boundaries don't align with bar boundaries. When you take one chord per bar from inferred, it might not match what a single-per-bar GT expects, especially with fine-grained segmentation (648 inferred chords vs 31 GT bars).

Root Cause Analysis

The real problem is NOT a format mismatch, but a segmentation mismatch.
  • GT: One chord per bar (or sometimes multiple per bar)
  • Inferred: 648 chords for a 7+ minute track = ultra-fine segmentation (~20 chords per bar on average)
  • Comparison: When merging inferred→per-bar for comparison, the averaging breaks down
