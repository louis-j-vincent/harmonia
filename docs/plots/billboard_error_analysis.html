Billboard v2 models — honest error analysis

Song-grouped 5-fold CV on 886 songs / 109,800 chords. McGill NNLS chroma, oracle GT boundaries. These are labelling-given-correct-timing numbers, not end-to-end.

Headline finding. The shipped corpus was built with chord_type="majmin", which collapses every dom / hdim / dim chord into maj/min/N. The "5-way quality head" therefore saw zero dom, hdim, or dim examples (maj 83,638 · min 26,162 · dom 0 · hdim 0 · dim 0). Its 77.9% CV accuracy ≈ the 76.2% majority-class floor: it is a maj-predictor, not a quality classifier.

Root confusion — the real, interpretable signal

Root CV acc = 0.762 ± 0.016 (matches the headline 76.5% — this number is honest). Top off-diagonal confusions are almost all ±5 / +7 semitones = the fourth / fifth:

Hypothesis. A major triad and its dominant share 2 of 3 pitch classes (C-E-G vs G-B-D share G; the fifth relationship dominates chroma overlap), and mean-pooled chroma over a segment mixes in the following chord's bass. Root errors are structured fifth/fourth slips, not noise — a bass-register prior or a transition/key prior would target exactly this (cf. the POP909 oracle sprint: bass evidence took root 53%→83%).

What this means

  • Root model (76.2%): real and useful — but on McGill NNLS chroma at oracle boundaries, a different feature space than the production Basic-Pitch pipeline (48-dim root-shifted BP chroma).
  • Quality head (v2): not usable as shipped — trained on collapsed GT with no dom/dim/hdim.
  • Billboard as a jazz teacher: even with full GT, dom ≈ 5% and dim/hdim < 0.3% of pop/rock chords — a weak teacher for the V7-vs-Imaj7 distinction that matters most in jazz.
