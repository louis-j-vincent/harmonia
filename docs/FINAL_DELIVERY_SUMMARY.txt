ARCHITECTURE SUMMARY
================================================================================

Integration Pattern:
  Harmonia Pipeline (P.chords)
    ↓ [Adapter function]
    ↓ Handoff format (BARS, SONG, etc.)
    ↓ [Engine: build() → DOM]
    ↓ Rendered UI (3 modes, 4 tabs, etc.)
    ↓ User interaction
    ↓ [POST /api/reinfer or /api/annotations]
    ↓ Server returns diff or persists
    
Key Insight:
  • Handoff engines are self-contained (vanilla JS, no build)
  • Data shapes already compatible (demo ≈ pipeline output)
  • Integration = data mapping, not reimplementation
  • Migration automated (migrate_annotator_tool.py)

Critical Success Factors:
  1. Use adapter functions (P → handoff format)
  2. Mount engines to correct DOM elements
  3. Wire real data (not demo data)
  4. Test at 390×844 vs. demo visually
  5. Run migration after template edits
